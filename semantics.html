But this defeats an important convention of web design: semantics. Ideally, the HTML for a page would contain nothing but markup. Each component of the page would be "marked up" with its type, like <article>, <header>, or <button>. Classes should be sparsely applied, as each element is likely to look the same throughout the site. But when pages are marked up using obscene amounts of nested divs, the semantics of the page are completely lost. A <div> is simply a division, and provides no clue as to its content.
