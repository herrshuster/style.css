Style.css

Semantics

Style.css intends to change the way that you look at writing the web. At its base, it is a CSS framework. However the problem with most CSS frameworks is that they do not aid in writing better CSS - they only help get sites up more quickly. To understand what I mean, a brief definition of semantics: "the meaning of a word or phrase." Having sound semantics means that one word or phrase always means the same thing in the same context. The web is a visual language and so a header element should look like a header element. A nav>ul>li should look different from a ul>li, and so on. And while one of the core qualities of a living language is that usage will determine its structure, Style.css attempts to provide a framework for the ways that people are already using the HTM Language. Instead of using classes, Style.css styles elements according to their semantics. An element within an article tag will appear a certain way. By using the most logically constructed HTML, we can create patterns in CSS that will only target elements in their context. It also eliminates the need for many, many classes.

Typography

The typography component of this framework started off with a blog post that many designers will have seen: Composing the New Canon. In this article, Owen Gregory outlines the basics of a typographic scale based on musical ratios. The idea is continued by Viljami Salminen in his blog post on musical typography But we must have a way to specify which "key" the page is set in, because there are six notes that all need to have related ratios: the h1-h5 tags and the p tag, upon whose size everything is calculated. By leveraging the power of SASS, we can program all the ratios into a library of mixins.

We will set all our measurements in ems and rems. That way we can set the font-size of the html element to a percentage ratio. By doing this, we can set the base tone. A page set in "C" will have a root em (rem) of 16px, and the other keys will be set as a percentage of that.

The SASS file will take the user-entered values of key and quality and output a vertical rhythm. The line-heights of headers will be multiples of the base line-height, and their sizes musical ratios of the base size. These can be used site-wide or by assigning a chord class (e.g. class="e maj7") to a body element to achieve the feeling of movement through a website outlined in the first article on musical typography.

Header

The header is a significant component of this CSS framework. By wrapping navigation components in the header tag, and following tranditional code patterns, we can construct a fully responsive, fluid header that collapses to a mobile-friendly navigation menu. Currently the implementation requires no Javascript and is done in pure HTML and CSS (relies on the checkbox hack). The header>nav>ul element is styled with display: table, so you can add as many list elements as you want and the nav bar will adapt. These navigation links become stacked on mobile. (Future versions will support the nav>ul>ul>li pattern for dropdowns.)

The header relies on several different sections: section.masthead, a wrapper for centering the img (logo) and the h1, the title of the site; section.search, a search bar that collapses to a search icon on mobile; and section.user, space for navigation links that collapse into the navigation panel on mobile.
